# P0-05 管理端人工补发稳定操作键

## 目标

为人工加积分、扣积分、加经验引入稳定 `operationKey`，恢复幂等与审计能力。

## 范围

- DTO 增加 `operationKey`
- `bizKey` 改为基于 `operationKey` 生成
- 保证重试不重复落账

## 主要改动

- 管理端 UI 或调用方提交固定唯一键
- 日志、账本、操作记录统一串联同一次人工操作
- 保留现有账本唯一约束作为最终兜底

## 完成标准

- 同一 `operationKey` 重试只会落一次账
- 运营可以通过 `operationKey` 追踪一次人工操作

## 执行信息

- 优先级：`S1`
- 硬前置：无
- 软前置：`P0-01`
- 直接后置：后续人工补发、审计、回滚类任务
- 可并行：`P0-01`、`P0-02`、`P0-04`
