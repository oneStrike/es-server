# P2-C-03 `CHAT` outbox 域闭环

## 目标

补齐 `CHAT` 域的消费闭环，但不把聊天和通知强行拉成同一个模型。

## 范围

- 为 `CHAT` 域补消费逻辑
- 明确是否写 delivery
- 明确 ack 边界

## 主要改动

- 让 `message_outbox` 的 `CHAT` 域不再停留在声明层
- 区分聊天 ack 与通知 ack
- 只复用必要基础设施，不统一两套业务语义

## 完成标准

- `CHAT` 域从“定义存在”变成“链路闭环”
- 不因为复用 outbox 就把聊天与通知模型混成一层

## 执行信息

- 优先级：`S3`
- 硬前置：`P2-B-03`
- 软前置：`P2-C-01`
- 直接后置：chat 域消息闭环
- 可并行：`P2-C-02`
