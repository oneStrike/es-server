# P2-B-03 通知投递结果表

## 目标

让 outbox 具备最小投递结果追踪，而不是只看到待发送记录。

## 范围

- 新增 `notification_delivery`
- worker 投递后写结果
- 管理端展示失败原因与重试次数

## 主要改动

- 区分成功、失败、重试中等状态
- 记录渠道、失败原因、重试次数、最终结果
- 管理端提供排障入口

## 完成标准

- outbox 之后的投递结果可查询
- 运营能知道消息是“未投递”还是“投递失败”

## 执行信息

- 优先级：`S2`
- 硬前置：`P2-B-01`
- 软前置：无
- 直接后置：`P2-B-04`、`P2-C-03`
- 可并行：`P2-B-02`
