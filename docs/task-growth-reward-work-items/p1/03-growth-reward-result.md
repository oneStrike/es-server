# P1-03 任务奖励返回结构化结果

## 目标

让任务服务拿到明确的发奖结果，而不是只依赖日志判断。

## 范围

- 为任务奖励定义返回类型
- 区分成功、幂等命中、真实失败
- 汇总积分和经验两条落账结果

## 主要改动

- `tryRewardTaskComplete()` 改为返回结构化对象
- 返回值包含记录 ID、幂等命中标记、错误信息
- warning log 继续保留，但不再作为唯一反馈

## 完成标准

- 调用方能根据返回值更新 assignment 状态
- 幂等跳过和失败能被清楚区分

## 执行信息

- 优先级：`S1`
- 硬前置：无
- 软前置：`P1-01`
- 直接后置：`P1-02`
- 可并行：`P1-04`
