# P1-05 混合成长账本分页接口

## 目标

提供统一的成长时间线，而不是让用户和运营分别查积分页、经验页。

## 范围

- App 端新增统一账本分页
- 管理端新增统一账本分页
- 保留旧接口作为兼容能力

## 主要改动

- 统一返回 `assetType / delta / beforeValue / afterValue / ruleType / bizKey / context / createdAt`
- 不立即替换旧 points / experience 分页
- 聚合排序口径以时间线为主

## 完成标准

- App 与管理端都能一次性查看积分和经验流水
- 旧接口继续可用，不形成强制切换

## 执行信息

- 优先级：`S2`
- 硬前置：`P1-04`
- 软前置：无
- 直接后置：`P2-B-04`
- 可并行：无
