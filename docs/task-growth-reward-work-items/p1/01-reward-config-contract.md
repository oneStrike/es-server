# P1-01 任务奖励配置契约收敛

## 目标

让 `rewardConfig` 的配置能力与真实结算能力一致，先消除“配置了但不生效”的误导。

## 范围

- 明确当前只支持 `points`、`experience`
- 调整 DTO 示例
- 增加创建和更新任务的结构校验

## 主要改动

- 去掉或显式标记 `badgeCodes` 为暂不生效
- 校验奖励字段类型、取值范围和 0 值策略
- 非法字段进入配置时给出明确错误

## 完成标准

- 管理端无法再录入当前不支持的奖励字段
- 任务奖励配置不再误导运营

## 执行信息

- 优先级：`S1`
- 硬前置：无
- 软前置：`P0-01`
- 直接后置：`P1-02`、`P1-03`
- 可并行：`P1-04`
