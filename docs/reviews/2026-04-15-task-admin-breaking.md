# Admin 前端破坏性更新清单

## 说明

- 本文只列 Task 模块中 Admin 前端会直接感知的合同变化。
- 本轮按破坏性更新执行：不提供兼容字段，不保留旧口径，不做 versioning。

## 1. 任务对账页最近事件语义切换

- 影响接口：
  - `GET admin/task/assignment/reconciliation/page`
- 影响字段：
  - `latestEventCode`
  - `latestEventBizKey`
  - `latestEventOccurredAt`
- 新语义：
  - 以上 3 个字段现在统一表示“按 `eventOccurredAt` 选出的最近一次事件”
  - 若多条事件的 `eventOccurredAt` 相同，后端再按更大的日志 ID 兜底
  - 若一条事件有 `eventOccurredAt`、另一条没有，优先返回有值的那条
- 旧语义失效：
  - 不能再按“最后写入的事件日志”理解这 3 个字段
  - 晚到/补放的旧事件不会再覆盖真正较新的业务事件

## 2. 前端必须调整

- 对账页“最近事件”相关展示、筛选联想和排障文案，全部按“事件发生时间”理解
- 若前端此前把这 3 个字段当作“最后处理事件”或“最后落库事件”使用，需要同步改成业务时间口径
- 不要再依赖旧实现中“晚写入旧事件会覆盖展示结果”的行为

## 3. 不变项

- 接口路径、请求参数和响应字段名保持不变
- 本次不新增兼容字段，也不删除现有字段
- 奖励提醒对账字段和任务基础字段不受本次变更影响
