## 全局改造原则

- 所有混合入口最终都要拆成 `contracts`、`core`、`module` 三类受支持子入口。
- DTO / Swagger / Controller 契约相关导出归入 `contracts`。
- Service / Resolver / Worker / runtime helper 归入 `core`。
- 仅供 Nest `imports` 装配的模块类归入 `module`。
- 历史命名为 `core`、`contract`、`module` 但职责混合的入口，也在本次改造范围内。
- 后续迁移任务应按本清单逐项勾选，不建议跳过全局前置项。

## 说明

- `split` 表示同一个 import 语句里很可能同时拿了 DTO 与 runtime 符号，后续改造时应拆成多条 import。
- `manual` 表示仅凭文件类型和命名无法安全判断，后续任务要人工确认。
- 本清单是全量扫描结果，不代表必须按一个 PR 一次性完成；但所有列出的混合入口都属于规范范围内的问题。
