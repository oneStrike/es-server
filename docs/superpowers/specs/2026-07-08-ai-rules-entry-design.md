# AI 规范快速入口设计

## 背景

当前仓库的规范主干已经比较完整：

- 项目级最小约束位于 `AGENTS.md`
- 规范索引位于 `.trae/rules/PROJECT_RULES.md`
- 专项规则拆分在 `.trae/rules/01-08*.md`

问题不在于规则缺失，而在于 AI 首次进入仓库时，仍需自行判断“本次改动属于哪一类”“需要先读哪些规则”“哪些现实差距是已知例外而不是本次任务必须顺手修复的内容”。

## 目标

为 AI 提供一层轻量、稳定、可快速检索的规范入口，降低首次执行成本，同时不新增第二套事实源。

## 设计原则

1. 新入口只做导航、释义和例外汇总，不重写专项规范。
2. 规范事实源仍然是 `AGENTS.md`、`.trae/rules/PROJECT_RULES.md` 和 `.trae/rules/01-08*.md`。
3. AI 进入仓库后，应能在一屏内判断本次改动要先看哪些规则、最低要做哪些验证。
4. 新入口必须显式区分“规则要求”和“当前仓库已知现实差距”。

## 信息架构

在 `.trae/rules/` 下新增三份 AI 辅助文档：

- `AI_RULE_ROUTER.md`
  - 第一站入口
  - 负责改动类型决策树、规则速查表、组合改动提示、执行顺序
- `AI_TERMS.md`
  - 统一解释项目内高频术语
  - 例如 `owner 文件`、`owner 模块`、`稳定 contract`、`闭集字段`
- `AI_EXCEPTIONS.md`
  - 汇总当前仓库已知的配置差距、流程约定和现实例外
  - 说明哪些问题已被识别，但默认不属于每次任务都必须顺手修复的范围

同时更新两个现有入口文件：

- `AGENTS.md`
  - 新增一段简短的 AI 快速入口提示
- `.trae/rules/PROJECT_RULES.md`
  - 在规则索引前新增 AI 快速入口区块

## 文档职责边界

### `AI_RULE_ROUTER.md`

必须回答四个问题：

1. 我这次改的是哪一类内容？
2. 我必须先读哪些专项规则？
3. 这种改动最容易犯什么错？
4. 最低验证应该做到什么程度？

文档结构采用：

1. 边界声明
2. 文本式决策树
3. 速查表
4. 组合改动提示
5. 固定执行顺序

### `AI_TERMS.md`

只解释项目内术语，不扩展为新的风格指南。每个术语采用统一模板：

- 术语
- 本仓库中的含义
- 典型例子
- 常见误解

### `AI_EXCEPTIONS.md`

只收录“已知且当前有效”的现实例外，不收录未来愿景。每项例外采用统一模板：

- 例外项
- 当前状态
- 关联规则
- 对 AI 的执行含义
- 何时需要显式升级处理

## 关键内容设计

### 决策树

采用文本决策树，不使用图形化流程图。原因：

- 维护成本更低
- diff 更清晰
- 更适合 AI 直接检索和引用

### 速查表

速查表至少包含以下列：

- 改动类型
- 必须先读
- 常见误判
- 最低验证
- 相关例外

其中“常见误判”是核心列，用来减少只贴链接但不提醒高频误用的问题。

### 组合改动提示

显式覆盖以下高频组合：

- Controller + DTO
- DTO + Type
- DTO + 错误语义
- Schema + DTO + Migration
- 文档 / 规则改动 + 仓库级验证

目标是减少 AI 只读单篇规则便开始改动的情况。

## 非目标

本轮不做以下事情：

1. 不改写 01-08 专项规则正文结构
2. 不新增机器可读规则清单
3. 不引入新的 lint、hook 或自动校验脚本
4. 不把所有规则重新复制到 AI 入口文档中

## 交付范围

本轮交付应包含：

- 新增 `.trae/rules/AI_RULE_ROUTER.md`
- 新增 `.trae/rules/AI_TERMS.md`
- 新增 `.trae/rules/AI_EXCEPTIONS.md`
- 更新 `AGENTS.md`
- 更新 `.trae/rules/PROJECT_RULES.md`

## 验证

本轮属于规则 / 规范文档改动，验证基线按 `AGENTS.md` 执行：

1. 对改动的 Markdown 文档执行 `pnpm exec prettier --check`
2. 再执行一次 `pnpm type-check`

## 风险与控制

- 风险：AI 入口文档变成第二套规范正文
  - 控制：每份辅助文档开头显式声明“仅作导航 / 释义 / 例外汇总”
- 风险：例外表变成历史垃圾堆
  - 控制：只收录当前有效且可执行解释的例外
- 风险：入口文档过长，失去“快速入口”价值
  - 控制：正文只保留高密度路由信息，不复制专项规则细节
