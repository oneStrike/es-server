# 签到模块改造工作包

## 文档职责

- 本目录承载签到模块从“滚动周期 + 统一基础奖励”重构为“自然周期 + 按日奖励 + 排期切换”的工作包文档集。
- 排期、依赖、状态仅在 [execution-plan.md](./execution-plan.md) 维护。
- 单任务文档只描述本任务的目标、范围、代码锚点、主要改动和完成标准，不维护第二套排序。
- 验收结论与证据统一沉淀到 [final-acceptance-checklist.md](./checklists/final-acceptance-checklist.md)。

## 已确认业务口径

- 任一自然日只允许存在一个生效签到计划。
- 允许切换计划，但不做立即切换；新计划只能从未来自然周期边界开始生效。
- `weekly` 表示真实自然周（周一到周日），`monthly` 表示真实自然月（每月 `1` 号到月末）。
- `startDate / endDate` 表示计划生效窗口，不再充当周期滚动锚点。
- 为避免半周期语义歧义，本轮默认要求计划边界与周期边界对齐：
  - 周计划 `startDate` 必须为周一，`endDate` 若存在则必须为周日。
  - 月计划 `startDate` 必须为每月 `1` 号，`endDate` 若存在则必须为月末。
- 基础奖励不再使用统一 `baseRewardConfig`，改为按 `dayIndex` 配置的 `dailyRewardRules`。
- 连续奖励继续独立于计划主表，但仍归属于 `planVersion`，且只在当前周期内计算。
- 计划未设置结束日期时，只表示持续生效；进入下一个自然周 / 月后，连续奖励、补签次数和周期统计重新开始，不跨周期累计。
- 补签只能发生在当前周期内、今天之前、且尚未签到的自然日，并继续受每周期补签次数限制。
- 本轮功能尚未对前端接入，也不存在需要兼容的历史签到数据。
- 本工作包按重设计处理，不以兼容旧 schema、旧 DTO 或旧奖励语义为目标。

## 本轮范围

- 重建签到计划、周期、记录和奖励规则的数据模型。
- 新增按日奖励规则 owner，并将其冻结进周期快照。
- 调整管理端计划定义接口，支持未来计划排期与非立即切换。
- 调整管理端对账 / 补偿链路，以及 App 端签到执行、补签、摘要和日历读模型。
- 补齐针对自然周期、排期切换、按日奖励与连续奖励的自动化验证。

## 非本轮范围

- 不支持同一天内即时切换生效计划。
- 不支持跨周期累计连续奖励。
- 不支持跨周期补签。
- 不兼容历史签到数据，也不做历史数据回填。
- 不为前端保留旧 DTO 字段兼容层。
- 不扩展为多生效计划并存模型。
