# P1-01 补齐验证、自动化测试与交付证据

## 目标

- 为本轮签到改造建立最小但完整的自动化回归保护和验收证据。
- 确保自然周期、按日奖励、排期切换和连续奖励重置不只停留在设计口径上。

## 范围

- 签到模块相关 spec 文件
- 变更文件的 `eslint`
- `pnpm type-check`
- `pnpm db:comments:check`
- [final-acceptance-checklist.md](../checklists/final-acceptance-checklist.md)

## 当前代码锚点

- 当前模块尚缺少围绕“自然周期、未来排期切换、按日奖励解析、连续奖励跨周期重置”的成体系回归保护。
- 当前工作包验收清单仅完成建档，尚未沉淀命令输出与证据。

## 非目标

- 不在本任务中继续扩展业务范围。
- 不在本任务中新增与本轮改造无关的测试资产。
- 不在本任务中引入只用于一次性验证的临时脚本并长期保留。

## 主要改动

- 补齐或新增以下自动化测试：
  - 计划日期边界对齐与排期窗口不重叠校验
  - 非立即切换和未来计划生效读取
  - 自然周 / 月周期解析与 `dayIndex` 推导
  - 按日奖励解析、签到记录奖励快照写入与账本结算
  - 连续奖励同周期触发、跨周期重置、计划切换后重置
  - App 端摘要、日历、动作返回契约
- 对涉及文件执行 `eslint`、`type-check` 与数据库注释校验，并把输出沉淀到验收清单。
- 按模块维度整理最终验收证据，明确是否存在阻塞上线项。

## 完成标准

- 本轮关键语义均有自动化测试或可复核命令输出支撑。
- `final-acceptance-checklist.md` 中的核心项已填写可复核证据，不存在“只有结论没有证据”的空缺。
- 未发现阻塞上线项时，有明确验证依据；若存在阻塞项，已记录原因、影响范围与解除条件。

## 完成后同步文档

- 更新 [execution-plan.md](../execution-plan.md) 中 `P1-01` 的状态。
- 回填 [final-acceptance-checklist.md](../checklists/final-acceptance-checklist.md) 的证据与结论。
