# 论坛主题通知改造最终成果验收清单

## 1. 文档目标

本文只保留“跨任务、跨模块”的统一验收项。

它不是：

- 阶段计划
- 开发排期版
- 单任务完成标准汇总表

使用方式：

1. 各任务自己的完成标准，以对应 work-item 文件为准
2. 联调、回归、准备上线时，再按本文核对跨任务事项
3. 每一项都要有代码、接口、测试或文档证据

## 2. 单任务验收事实源

每个任务自己的目标、范围、完成标准、非目标、代码锚点，以对应 task 文件为准：

- `docs/forum-topic-notification-work-items/p0/*`
- `docs/forum-topic-notification-work-items/p1/*`
- `docs/forum-topic-notification-work-items/p2/*`

本文不再重复抄写各任务的细项完成标准，避免多份清单漂移。

## 3. 总体验收结论标准

以下条件全部满足，才能认定“论坛主题通知改造验收通过”：

- [ ] 论坛主题点赞、收藏、评论、评论回复四类通知语义清晰可区分
- [ ] 论坛主题点赞与收藏不再复用 `COMMENT_LIKE / CONTENT_FAVORITE`
- [ ] outbox 通知类型来源已收口，不再双写漂移
- [ ] 动态模板与业务 fallback 能同时工作
- [ ] 论坛主题通知文案能表达“谁对哪条主题做了什么”
- [ ] 核心链路有自动化测试覆盖，且主要测试通过
- [ ] 文档、代码、seed、DTO 口径一致

说明：

- 本轮验收目标是把论坛主题通知做成可复用样板，不是要求所有通知都改成论坛互动式标题 / 正文。
- 若后续推广到全通知域，应复用同一套治理机制；系统公告、任务提醒、聊天消息等类型可以保留各自更合适的文案模型。
- 本轮签收默认要求四类论坛主题通知一起收口；如果只完成其中一部分，不建议按“论坛主题通知改造完成”签收。
- `COMMENT_REPLY` 本轮验收只要求动态化与可读兜底，不以论坛主题 / 作品 / 章节域级拆分为验收条件。

## 4. 数据与兼容性验收

- [ ] 新增通知类型后，不会导致旧通知查询报错
- [ ] 旧通知类型 `smallint` 编码保持不变，新类型只追加不重排
- [ ] 模板 seed 已包含新增 `TOPIC_*` 类型
- [ ] 旧通知数据不需要回写也能继续展示
- [ ] App / 管理端能识别新增 `TOPIC_*` 类型
- [ ] `COMMENT_REPLY` 升级为动态文案后，旧回复通知仍可正常查看

验收证据：

- [ ] seed 抽查结果
- [ ] 通知分页接口抽查结果
- [ ] App 与管理端类型展示抽查结果

## 5. 接口与兼容性验收

- [ ] 新增通知类型已同步到 DTO
- [ ] 新增或调整的通知 payload 字段有明确说明
- [ ] 旧接口若保留，行为未被静默破坏
- [ ] 模板缺失 / 禁用 / 渲染失败时，通知仍可正常落库
- [ ] 已启用模板环境下，`COMMENT_REPLY` 也能展示动态文案
- [ ] App 通知列表对新增标题 / 正文口径展示正常

验收证据：

- [ ] DTO 抽查结果
- [ ] 通知创建与通知分页 smoke test 通过

## 6. 可观测性与幂等验收

- [ ] 主题点赞、收藏、评论、评论回复各自都有稳定幂等键
- [ ] delivery 能区分新增主题通知类型
- [ ] 自通知跳过、偏好关闭、幂等跳过与真实失败可以区分
- [ ] outbox 技术状态与业务投递状态没有被混为一谈

验收证据：

- [ ] 至少抽查主题点赞、主题收藏、主题评论、评论回复各一条完整链路

## 7. 测试与质量验收

- [ ] `notification-template.service` 有缓存与校验测试
- [ ] `comment.service` 有评论回复动态文案测试
- [ ] `forum-topic-like.resolver` 有 `TOPIC_LIKE` 测试
- [ ] `forum-topic-favorite.resolver` 有 `TOPIC_FAVORITE` 测试

### 7.2 命令验收

- [ ] `pnpm type-check` 通过
- [ ] 相关测试命令通过
- [ ] 新增 spec 不依赖脆弱的时间戳或随机顺序

### 7.3 回归重点

- [ ] 论坛主题收藏主链路未回归
- [ ] 论坛主题评论主链路未回归

## 8. 文档与交付验收

- [ ] 设计文档已更新
- [ ] 执行计划文档已更新
- [ ] 开发排期版已更新
- [ ] 单任务文档已更新
- [ ] 最终成果与文档描述一致
- [ ] 文档中不存在已知过期口径

验收证据：

- [ ] 抽查设计文档、任务文档、接口 DTO，口径一致
- [ ] 新成员只看文档也能区分四类论坛主题通知

## 9. 阻塞上线项

以下任一项未通过，都建议视为阻塞上线：

- [ ] 论坛主题点赞仍复用 `COMMENT_LIKE`
- [ ] 论坛主题收藏仍复用 `CONTENT_FAVORITE`
- [ ] 论坛主题一级评论仍不会触发“主题被评论”通知
- [ ] 评论回复通知仍然只有固定文案，没有动态正文
- [ ] 模板异常会导致通知无法落库
- [ ] 新增通知类型没有基础测试覆盖

## 10. 最终签收标准

当以下问题都能回答“是”时，可以签收：

- [ ] 我们能否准确区分主题点赞、主题收藏、主题评论、评论回复四类通知？
- [ ] 我们能否从通知类型、模板、偏好、delivery 四个维度解释论坛主题通知？
- [ ] 我们能否保证模板异常时仍然看到正确 fallback 文案？
- [ ] 我们能否保证一级评论与回复评论不会互相误发通知？
- [ ] 我们能否让用户一眼看出“是谁对哪条主题做了什么”？
- [ ] 我们能否让新同事只看文档就不把主题点赞和评论点赞混为一类？

只要以上问题全部回答“是”，这轮任务才算真正完成。
