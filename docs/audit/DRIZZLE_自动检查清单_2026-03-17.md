# Drizzle 自动检查清单（项目级）

适用范围：本仓库所有使用 Drizzle 的模块与服务。  
目标：提供一份“可自动化执行”的检查清单，用于 CI 或人工批量核查。  
更新日期：2026-03-17

## 1. 执行顺序（建议）

1. 代码扫描（禁用项 + 迁移残留）
2. 规范扫描（写入语义 / 分页 / 事务）
3. 原生 SQL 与类型逃逸扫描
4. 规则一致性与验收
5. 全量校验（lint + type-check）

## 2. 代码扫描（禁用项 + 迁移残留）

以下命中必须为 0：

- `this.prisma`
- `extends PlatformService`
- `Prisma.`（类型或运行时）
- `PrismaTransactionClientType` / `PrismaClientType`

```powershell
rg -n "this\.prisma" .\libs .\apps .\db
rg -n "extends\s+PlatformService" .\libs .\apps .\db
rg -n "Prisma\." .\libs .\apps .\db
rg -n "Prisma(TransactionClientType|ClientType)" .\libs .\apps .\db
```

## 3. 规范扫描（写入语义 / 分页 / 事务）

### 3.1 写操作规范

- 写操作必须 `withErrorHandling` 包裹。
- update/delete 必须 `assertAffectedRows`。

快速巡检（人工复核）：

```powershell
rg -n "\.insert\(|\.update\(|\.delete\(" .\libs .\apps
rg -n "withErrorHandling" .\libs .\apps
rg -n "assertAffectedRows" .\libs .\apps
```

判定建议：

- 若某文件包含 `.insert/.update/.delete` 却未出现 `withErrorHandling`，需人工复核。
- 若存在 `.update/.delete` 却未出现 `assertAffectedRows`，需人工复核。

### 3.2 分页规范

- 列表分页必须使用 `drizzle.ext.findPagination`。
- `pageIndex` 统一 1-based。

```powershell
rg -n "findPagination" .\libs .\apps
rg -n "pageIndex\s*=\s*0" .\libs .\apps
```

### 3.3 事务规范

- 禁止 `tx: any`、`void tx`。
- 事务必须传递且被使用。

```powershell
rg -n "tx:\s*any" .\libs .\apps
rg -n "void\s+tx" .\libs .\apps
```

## 4. 原生 SQL 与类型逃逸扫描

规则：

- 允许 `sql\`...\`` + `db.execute`，但必须下沉到 `db/core/query`。
- 禁止业务层 `(result as any).rows` 手工解包。

```powershell
rg -n "db\.execute\(sql`" .\libs .\apps
rg -n "\.rows\b|\(.*as any\)\.rows" .\libs .\apps
rg -n "sql`" .\libs .\apps
```

判定建议：

- `db.execute(sql\`...\`)` 若不在 `db/core/query` 目录，需人工复核。
- 任何 `.rows` 解包应视为违规，必须重构为强类型返回。

## 5. 规则一致性与验收

以下为“必须通过”的最终验收点：

1. 代码中不再出现 `this.prisma`、`extends PlatformService`、Prisma 类型透出。
2. 写操作均具备 `withErrorHandling`，更新/删除具备 `assertAffectedRows`。
3. 分页统一 `findPagination`，且 `pageIndex` 1-based。
4. 事务类型无 `any`，且无 `void tx`。
5. 原生 SQL 已收敛至 `db/core/query`，业务层无 rows 解包。

## 6. 全量校验

```powershell
pnpm type-check
pnpm lint
```
