# 验证与临时测试规范

适用范围：开发过程中的本地验证、临时测试文件、探针脚本与交付前检查。

## TL;DR

- 何时看：改验证方式、临时测试、探针脚本、交付前检查要求时先看本篇。
- 必做：行为变化必须补足够的验证证据；临时测试 / 探针用完即删；文档、规则和代码改动都要跑对应基线检查。
- 不要：保留测试文件，不要把 `pnpm test`、`pnpm test:e2e` 当默认交付命令，也不要没跑验证就声称完成。
- 最低验证：文档改动至少跑对应 Markdown 检查；规则文档再加 `pnpm type-check`；代码改动至少 `pnpm type-check`。

## 仓库约定

- 仓库不保留任何测试文件；`*.spec.ts`、`*.test.ts`、`*.e2e.spec.ts`、`__tests__/`、`test/`、`tests/` 只能作为临时验证资产存在。
- 开发中为复现问题或验证改动临时创建测试文件时，必须在完成验证后、交付前删除。
- 临时测试文件不是长期资产，不作为代码审查、回归保护或提交内容保留。
- `pnpm test`、`pnpm test:cov`、`pnpm test:e2e` 不作为默认交付验证命令，也不能作为保留测试文件的理由。

## 默认动作

- 改动涉及行为、接口 contract、错误语义、事务语义、分页、时间逻辑、幂等、快照映射、计数器同步时，必须补充足够的验证证据。
- 优先使用 `pnpm type-check`、lint、build、静态分析、真实调用链检查或人工可复现步骤验证改动。
- 若上述方式不足以证明行为，可临时编写靠近改动点的测试或探针，运行后立即删除。
- 纯文档、纯注释、纯格式化变更通常不需要临时测试，但仍要跑必要的基线检查。
- cleanup / refactor / deslop 任务，默认先确认行为保护方式；需要回归验证时可临时补测试，完成后必须删除。

## 何时必须临时测试

- 新增或修改对外可见行为，且无法用类型检查、lint、build 或现有可运行路径证明。
- 调整错误码、异常类型、错误 message、降级策略，且需要可执行断言确认语义。
- 涉及事务、并发、重试、补偿、幂等、原子更新、计数同步，且存在回归风险。
- 调整分页、排序、筛选、默认值、时间窗口或时区语义，且需要边界样例证明。
- DTO / mapper / serializer / projection / notification payload 变更，且需要确认公开 contract。
- 修复回归 bug，且需要可复现步骤证明修复有效。

## 临时测试文件要求

- 临时测试应尽量靠近 owner 文件，命名清楚，覆盖当前验证目标即可。
- 临时测试只断言关键字段、错误码、状态变化和副作用；不要制造大而脆的快照。
- 与时间相关的逻辑要固定时间源或使用稳定输入，避免验证漂移。
- 与随机值、排序、分页相关的断言要写清预期顺序和边界。
- 对错误路径至少覆盖一种代表性失败场景，明确断言错误语义。
- 临时测试验证完成后，必须删除对应测试文件、探针脚本和一次性夹具。

## 默认验证命令

- 基线：`pnpm type-check`
- 规则敏感层变更：对改动文件运行必要的 `eslint` 或等价静态检查。
- 入口或构建链路变更：运行对应 app / package 的 build 命令。
- 临时测试验证：可运行 `pnpm exec jest --runInBand --runTestsByPath <temp-spec-path>`，随后删除临时测试文件。
- `pnpm test`、`pnpm test:cov`、`pnpm test:e2e` 不作为默认交付验证命令。

## 已知配置矛盾

- `package.json` 安装了 `jest`、`ts-jest`、`supertest`、`@types/jest`、`@types/supertest` 等测试依赖，但未定义 `test` 脚本，且仓库不保留测试文件。当前临时测试通过 `pnpm exec jest` 直接调用，不依赖 `pnpm test` 脚本。后续如需统一入口，应先补充 `test` 脚本或清理未使用的测试依赖；两者取其一，不应长期并存。

## 禁止项

- 禁止提交或长期保留任何测试文件、测试目录、临时探针、手工验证脚本或一次性夹具。
- 禁止在行为变更后完全不做验证。
- 禁止为了省事删除失败验证证据后直接声称完成；必须修正实现或重新选择有效验证方式。
- 禁止只断言实现细节，例如私有方法调用次数、局部变量中间值、无业务意义的内部步骤。
- 禁止在未运行验证命令的情况下声称“验证通过”“已修复”“已完成”。

## 正反例

- 允许：临时为 mapper 创建 `*.spec.ts`，断言公开返回结构与异常分支；运行后删除该文件。
- 允许：改动 service 的事务逻辑后，临时补一个覆盖计数同步和资源不存在分支的 spec；验证后删除。
- 允许：只改一个 mapper 时，运行 `pnpm type-check`，必要时加临时 spec 验证；交付前删除临时 spec。
- 禁止：改了分页默认排序，却只跑 `pnpm type-check` 且没有任何边界验证证据。
- 禁止：为了省事删除失败 spec 或探针输出，而不是修正实现或测试数据。
- 禁止：把整个大对象直接快照，导致一点无关字段变化就让验证失真。
